# Objective 

The objective of this scenario is to simulate a malicious webshell attack and evaluate the effectiveness of OSSEC as a Host Intrusion Detection System (HIDS) in detecting, monitoring, and analyzing unauthorized file modifications within a web server environment.

---
# Tools Used

- **OSSEC (HIDS - Host Intrusion Detection System)**  
  Security monitoring tool to detect file changes, generate alerts, and analyze system activity.
  
--- 

# Tasks – Web Shell Detection using OSSEC FIM
- **Attack simulation:** Inject a malicious script (webshell) into a web server directory.  
- **Analysis:** Inspect the alert log file to verify file detection by OSSEC.  
- **Custom rule creation:** Increase the severity of the rule that detects file changes.

---
# Executive summary:
In this report we simulate a webshell and use ossec as a Host Intrusion Detection System to detect and analyze malicious activity. First, we enable File Integrity Management (Syscheck) to monitor the system for newly created files. Next, we inject a shell.php file into the root web directory /var/www/html. Then, we analyze alert logs generated by ossec to detect malicious activity and verify the effectiveness of the ossec mechanism. Finally, we prioritize File Integrity Management FIM by increasing its severity through a custom rule defined in local_rules that match Rule ID 554 and overrides its default severity level.

## 2.3. Analysis:
Based on alerts recorded on alerts.log file, ossec successfully detected and monitored the system activities, confirms that detecting and monitoring mechanisms are working correctly.

Many alerts are generated including:

- Rule 31122 (severity level 5): internal web server error detected
- Rule 531 (severity level 7): critical disk space usage detected

These alerts are generated as a result of restarting ossec service using sudo systemctl restart ossec command.

The most important alert observed was related to the addition of shell.php file to the web root directory. This alert has the following characteristics:

- Rule ID:554
- Severity:5
- Type: File Integrity Management
- Location:/var/www/html/

This alert indicates that a new file named shell.php was added to /var/www/html/ directory and was detected with a medium severity level. Confirming that ossec correctly detected the malicious activity by comparing new changes to its baseline.

## 2.4. Hands on question:
Using Rule ID 554, we can prioritize the File Integrity Management FIM by increasing its severity level through a custom rule on local_rules.xml. This can be achieved by editing the local rules using nano /var/ossec/rules/local_rules.xml command and creating a rule that matches Rule 554 and overrides its severity to 11.

# Conclusion:
- The ossec successfully detects the creation of a new file (shell.php) into the web root directory(/var/www/html), which indicates that the FIM (syscheck) works correctly.
- The alert triggered Rule ID 554(new file added to the system) with a medium severity (5) indicates that ossec correctly classified the event as suspicious activity.
- Other alerts, disk space usage (Rule ID 531) and server web error (Rule ID 31122) indicates that ossec provides a valuable system and service monitoring.
- All alerts were recorded in alerts.log file enabling an effective inspection and forensic analysis for security events.
